🚀 Panduan Memulai Cepat

Langkah dasar untuk mengakses aplikasi dan memuat data pertama kali.

1
Buka Halaman Web

Saat pertama kali membuka website ini, Anda akan diarahkan ke halaman "Konfigurasi Awal" jika belum pernah melakukan setup di browser tersebut.

2
Masukkan URL Database

Masukkan Google Apps Script Web App URL yang didapat dari lembar kerja Google Sheets (tanyakan penanggung jawab database jika Anda bukan pembuatnya).

3
Masukkan Password & Hubungkan

Buat password pengaman (atau masukkan password tim yang sudah disepakati). Tekan tombol Simpan & Kunci. Sistem akan mengamankan URL tersebut di browser Anda.

4
Mulai Monitoring

Aplikasi akan memuat data Google Sheets secara real-time. Anda kini berada di dasbor monitoring utama dan siap bekerja!

📝 Manajemen Bahan Ajar

Panduan lengkap cara mengelola record, mengedit, dan memahami kolom-kolom data.

add_box Menambahkan Bahan Ajar Baru

Klik tombol "Tambah Data" di pojok kiri atas bilah navigasi. Isi formulir yang muncul (Judul dan Jenis Bahan Ajar wajib diisi). Kolom persentase pengerjaan akan otomatis terhitung berdasarkan status penulisan dan pengiriman Nota Dinas (ND) Honor.

edit Mengubah atau Menghapus Data

Klik judul bahan ajar di tabel utama untuk membuka Detail Informasi. Di dalam modal detail tersebut, klik "Edit Data" untuk mengubah isi atau klik "Hapus" untuk menghapusnya secara permanen dari server Google Sheets.

rule Arti Istilah & Kolom Utama
  • Status Penulisan & ND Honor: Mengukur persentase pengerjaan. Jika penulisan *Selesai* (+50%) dan ND Honor *Kirim/Tanpa Honor* (+50%), total persentase menjadi 100%.
  • Terbit & SK: Menyimpan tautan digital (Link Google Drive) dari Surat Keputusan (SK) terkait penerbitan bahan ajar.
  • Similarity (Kemiripan): Persentase tingkat kemiripan judul dengan buku lain di database. Bertujuan mendeteksi judul ganda/plagiasi secara otomatis.

🔒 Keamanan & Sinkronisasi Database

Penjelasan teknologi keamanan kriptografi dan konfigurasi Google Sheets.

Aplikasi ini dirancang sebagai website statis murni yang berjalan 100% di browser pengguna tanpa server backend khusus, namun dilengkapi keamanan sekelas aplikasi perbankan modern:

key Kunci Tanda Tangan Digital (SIGNATURE_KEY)

Setiap request kirim-terima data ditandatangani secara kriptografis menggunakan algoritma `HMAC-SHA256` menggunakan password Anda sebagai kunci. Google Sheets hanya akan memproses data jika tanda tangan digital tersebut valid dan cocok dengan yang terdaftar di Script Properties Google Sheets Anda.

history Kedaluwarsa Request & Nonce

Request data dilindungi dari penyadapan (Replay Attacks) dengan masa kedaluwarsa ketat 60 detik dan sistem `nonce` unik. Setiap request hanya berlaku sekali pakai dan langsung kedaluwarsa secara otomatis.

Jawaban cepat atas pertanyaan atau kendala operasional yang sering ditemui.

Cukup arahkan kursor (hover) Anda di atas persentase similarity (misal: `68%`) di tabel utama. Sebuah balon petunjuk (*tooltip*) akan muncul menunjukkan judul buku pembanding yang dinilai mirip. Selain itu, Anda juga dapat mengklik judul buku untuk membuka Detail Bahan Ajar, di mana nama buku yang mirip akan tertulis jelas di bawah skor similarity.

Lakukan setup awal kembali, lalu salin (copy) password Anda. 3. Masuk ke halaman Apps Script Google Sheets Anda, pilih **Project Settings** (ikon roda gigi), tambahkan Script Property baru bernama SIGNATURE_KEY dan tempelkan password tersebut sebagai nilainya. Simpan dan deploy ulang web app Anda.

Koneksi Aman Terbentuk!

Gunakan Password Anda sebagai kunci tanda tangan digital (SIGNATURE_KEY) untuk Google Apps Script Anda.

Instruksi Apps Script: 1. Buka Google Sheet > Extensions > Apps Script. 2. Buka Project Settings (ikon roda gigi di kiri). 3. Klik Add Script Property di bawah halaman. 4. Set nama property: SIGNATURE_KEY 5. Paste Password Anda (kunci di atas) sebagai value, klik Save.
